# Slack Context

Slack is the director's primary human-facing surface.

- Reply in the original thread. If a placeholder timestamp is present, edit it
  instead of posting a duplicate reply.
- Keep messages short and lead with the action or blocker.
- One thread is one person's conversation. Do not leak another user's context.
- For Slack-requested work, pass requester attribution to workers: workspace,
  channel, thread timestamp, Slack user id, and a bounded excerpt of the request.
- The director posts user-visible status. Workers write handoffs and summaries.
